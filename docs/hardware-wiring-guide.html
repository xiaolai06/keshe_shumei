三、各模块详细接线

1. DS18B20 温度传感器

🔌 接线方式
DS18B20 引脚接树莓派说明
VCC（红/左脚）3.3V（物理1）供电，也可接 5V
GND（黑/右脚）GND（物理6）接地
DQ（黄/中间脚）GPIO 4（物理7）数据线，必须接 4.7kΩ 上拉电阻到 3.3V
⚠ 必须接 4.7kΩ 上拉电阻：从 DQ 线拉到 3.3V，不接读不到数据。面向平面（有字的一面）引脚从左到右是 GND、DQ、VCC。 
# 开启 1-Wire（只需一次）
sudo nano /boot/config.txt
# 在末尾加入：
dtoverlay=w1-gpio

# 重启后检查
ls /sys/bus/w1/devices/
# 应该看到 28-xxxxxxxxxxxx 的文件夹

MCP3008 ADC 模块（光照 + 气体传感器共用）

🔌 接线方式（SPI 总线）
MCP3008 引脚接树莓派说明
VDD3.3V（物理1）数字供电
VREF3.3V（物理1）参考电压（决定 ADC 量程）
AGNDGND（物理6）模拟地
DGNDGND（物理6）数字地
CLKGPIO 11 (SCLK)（物理23）SPI 时钟
DOUTGPIO 9 (MISO)（物理21）数据输出（ADC → 树莓派）
DINGPIO 10 (MOSI)（物理19）数据输入（树莓派 → ADC）
CS/SHDNGPIO 7 (CE1)（物理26）片选（CE1，与 OLED 共享 SPI0 总线）
CH0MQ-135 气体传感器 SIG通道 0 → 气体浓度
CH1光照传感器 SIG通道 1 → 光照强度
💡 MCP3008 是 8 通道 10 位 ADC，通过 SPI 接口通信，片选用 CE1 (GPIO 7)。本项目使用 CH0 接气体传感器、CH1 接光照传感器。3.3V 参考电压下分辨率约 3.2mV/级。与 OLED 共享 SPI0 总线，靠不同片选区分。

MQ-135 气体传感器

🔌 接线方式（模拟输出）
MQ-135 引脚接树莓派说明
VCC5V（物理2）供电（需 5V，加热丝需要电流）
GNDGND（物理6）接地
AOUTMCP3008 CH0模拟输出（气体浓度 → 电压）
DOUT不接数字输出（本项目不用，使用模拟量）
⚠ MQ-135 需要预热 24~48 小时才能输出稳定数据。首次使用请先通电预热至少 2 分钟再读取。

2. DHT11 温湿度传感器

🔌 接线方式
DHT11 引脚接树莓派说明
VCC（+ / 左脚）3.3V（物理1）供电
DATA（中间脚）GPIO 17（物理11）数据线，必须接 10kΩ 上拉电阻到 3.3V
GND（- / 右脚）GND（物理6）接地
⚠ DHT11 读取间隔不能小于 2 秒，否则容易读取失败。面向有孔的一面，引脚从左到右是 VCC、DATA、GND（有的模块是4脚，中间两个是DATA，只接一个）。

3. 光照传感器（模拟模块）

🔌 接线方式（模拟输出，需 ADC）
光照传感器引脚接树莓派说明
VCC3.3V（物理1）供电
GNDGND（物理6）接地
SIGMCP3008 CH1模拟信号输出，需通过 ADC 转换后才能被树莓派读取
⚠ 树莓派没有模拟输入引脚，光照传感器的 SIG 输出必须先经过 MCP3008 ADC 模块转换为数字信号，再通过 SPI 传给树莓派。MCP3008 的接线见下方 ADC 说明。
💡 你的光照模块只有 3 个引脚（VCC、GND、SIG），是模拟输出型。SIG 输出的电压随光照强度线性变化：光照越强，电压越高。如果你希望直接获取 lux 值，可以考虑换成 I2C 数字光照模块 BH1750（GY-302），它有 5 个引脚（VCC、GND、SDA、SCL、ADDR），直接输出 lux，无需 ADC。

4. SSD1306 OLED 显示屏（0.96 寸）

🔌 接线方式（SPI 模式，与 MCP3008 共享 SPI0 总线）
SSD1306 引脚接树莓派说明
VCC3.3V（物理1）供电（有的模块支持 5V）
GNDGND（物理6）接地
DINGPIO 10（物理19）SPI 数据输入（MOSI，与 MCP3008 共用）
CLKGPIO 11（物理23）SPI 时钟（SCLK，与 MCP3008 共用）
CSGPIO 8（物理24）片选（CE0）
DCGPIO 13（物理33）数据/命令选择（高=数据，低=命令）
RSTGPIO 24（物理18）复位（拉低复位，正常工作时拉高）
💡 OLED 与 MCP3008 共享 SPI0 总线（MOSI/MISO/SCLK），靠不同片选引脚区分：OLED 用 CE0 (GPIO 8)，MCP3008 用 CE1 (GPIO 7)。同一时刻只能有一个设备在通信，代码中需注意切换。

5. RGB LED（共阳极）

🔌 接线方式
LED 引脚接树莓派说明
R 红GPIO 27（物理13）串联 330Ω红色通道
G 绿GPIO 22（物理15）串联 330Ω绿色通道
B 蓝GPIO 5（物理29）串联 330Ω蓝色通道
最长脚 / VCC3.3V（物理1）公共阳极
⚠ 每个颜色必须串联一个 330Ω 限流电阻，否则通电瞬间电流过大会烧坏 LED。长脚是公共脚，共阳极接 3.3V。注意：共阳极的代码逻辑与共阴极相反 — GPIO 输出 HIGH = 灭，GPIO 输出 LOW = 亮，PWM 占空比需要反转。

6. 有源蜂鸣器

🔌 接线方式
蜂鸣器引脚接树莓派说明
正极 (+) / 红线GPIO 12（物理32）信号输入
负极 (-) / 黑线GND（物理6）接地
💡 有源蜂鸣器给高电平就响，给低电平就停；无源蜂鸣器需要 PWM 方波才能发出不同频率的声音。如果买的蜂鸣器模块有两个引脚，通常是已经内置了驱动电路。

9. INMP441 I2S 麦克风（×2 个）

🔌 接线方式（立体声双麦）
INMP441 引脚接树莓派说明
VDD3.3V（物理1）供电（两个都接）
GNDGND（物理6）接地（两个都接）
SCK (BCLK)GPIO 18（物理12）位时钟（两个共用）
WS (LRCK)GPIO 19（物理35）字选择 / 左右声道时钟（两个共用）
SD (DIN)GPIO 20（物理38）数据输出（两个都接 GPIO 20）
L/R第一个接 GND，第二个接 3.3VL/R 接 GND = 左声道输出，接 VCC = 右声道输出
⚠ L/R 引脚是关键：一个麦克风接 GND（左声道），另一个接 3.3V（右声道），这样树莓派才能区分两个麦克风的数据。 
# 启用 I2S（只需一次）
sudo nano /boot/config.txt
# 在末尾加入：
dtoverlay=i2s-mmic

# 重启后检查录音设备
arecord -l
# 应该看到 I2S 设备

七、注意事项

  1. 接线前断电：所有接线操作必须在树莓派断电状态下进行
  2. 先检查短路：通电前检查所有接线，确认没有短路（特别是 3.3V 和 GND）
  3. SPI 总线共享：SSD1306 OLED 和 MCP3008 ADC 共享 SPI0 总线（MOSI/MISO/SCLK），OLED 用 CE0 (GPIO 8)，MCP3008 用 CE1 (GPIO 7)，同一时刻只能有一个设备通信
  4. 1-Wire 上拉必须有：DS18B20 的 4.7kΩ 上拉电阻是必须的，不能省略
  5. LED 必须限流：RGB LED 每个颜色串联 330Ω 电阻，不能直接接 GPIO
  6. RGB LED 共阳极：公共脚接 3.3V，GPIO 输出 LOW = 亮、HIGH = 灭，代码中 PWM 占空比需要反转
  7. 光照传感器需 ADC：模拟光照传感器的 SIG 引脚必须接 MCP3008 ADC，不能直接接树莓派 GPIO
  8. DHT11 等待时间：两次读取间隔至少 2 秒
  9. GPIO 编号：代码里用的是 BCM 编号（GPIO 4/17/27...），不是物理引脚编号
