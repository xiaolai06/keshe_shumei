二、开发计划（4 周）

第 1 周 · 硬件驱动 + 基础架构

目标：让树莓派"活"起来
  1. 传感器驱动 — DS18B20 温度 + DHT11 湿度 + BH1750 光照，读取真实数据替换 mock
  2. OLED 驱动 — SSD1306 I2C 通信，实现表情切换 + 文字滚动
  3. RGB LED — GPIO PWM 控制，7 种情绪对应 7 种颜色
  4. 蜂鸣器 — PWM 提示音：提醒、欢迎、升级
  5. SQLite 初始化 — 运行 schema.sql 建表，WAL 模式
  6. 传感器轮询 — APScheduler 每 5 秒采集，存入 sensor_readings 表

交付物：传感器数据可在终端打印，OLED 显示表情，LED 随温度变色

第 2 周 · Agent + 记忆 + 对话

目标：能聊天、能记住
  1. DeepSeek API 接入 — OpenAI 兼容接口，system prompt + 多轮对话
  2. LangGraph 状态机 — 实现 Perceive → Recall → Think → Decide → Act → Reflect 6 个节点
  3. 工具集 — set_reminder / search_memory / set_oled / set_led / play_buzzer / take_photo
  4. 三层记忆
    • 工作记忆：内存中最近 10 轮对话
    • 短期记忆：SQLite 今日摘要，保留 7 天
    • 长期记忆：SQLite + ChromaDB 向量检索
  5. 情绪计算 — 4 维度加权（环境/交互/时间/异常），EMA 平滑，3 次一致才切换
  6. Web 对话接口 — POST /chat + SSE 流式返回，前端实时显示打字效果

交付物：通过网页与桌宠对话，桌宠能记住偏好，情绪影响回复风格

第 3 周 · 多模态 + Web 全功能

目标：看得见、听得到
  1. 摄像头 — OpenCV 采集 + MJPEG 流推送到前端
  2. 人脸识别 — face_recognition 或 dlib，识别主人并触发记忆
  3. 语音识别 — INMP441 I2S 双麦采集 + Whisper tiny 本地推理
  4. 语音合成 — edge-tts 在线 TTS + USB 音响播放
  5. Web 全功能 — 传感器图表接入真实数据 / 记忆搜索 / 提醒 CRUD / 摄像头拍照分析
  6. WebSocket — 传感器实时推送 + 情绪变化通知 + 新消息推送

交付物：摄像头画面实时显示，语音对话可用，前端数据全部为真实数据

第 4 周 · 联调 + 优化 + 答辩

目标：稳定运行、准备展示
  1. 全模块联调 — 传感器 → Agent → 硬件输出 → Web 展示完整链路
  2. OLED 表情素材 — 制作 7 种情绪的 128x64 位图
  3. 性能优化 — Whisper 模型预加载 / Chart.js 数据节流 / 图片压缩
  4. 异常处理 — 传感器断线降级 / LLM 不可用时关键词匹配 / 摄像头断开重连
  5. 答辩准备 — 演示脚本 / 功能亮点梳理 / 常见问题准备

交付物：稳定运行的完整系统，可流畅演示全部功能

三、技术要点

API 接口规范

// 统一返回格式
{
    "success": true,
    "data": { ... },
    "error": null
}

// 对话接口（SSE 流式）
POST /api/chat        → JSON 单次回复
POST /api/chat/stream → SSE 流式 token

// 传感器
GET  /api/sensors           → 实时数据
GET  /api/sensors/history   → 历史趋势

// 摄像头
GET  /api/camera/stream     → MJPEG 流
POST /api/camera/capture    → 拍照分析

// 记忆
GET  /api/memory            → 长期记忆列表
GET  /api/memory/search?q=  → 语义搜索

// 提醒
GET    /api/reminders       → 列表
POST   /api/reminders       → 创建
DELETE /api/reminders/{id}  → 删除

数据库表

表名用途关键字段
sensor_readings传感器历史temp, humidity, light, timestamp
interactions对话记录role, content, mood, timestamp
short_term_memory短期摘要summary, topics, date_range
long_term_memory长期记忆content, category, importance, embedding_id
reminders提醒type, cron_expr, content, enabled
mood_log情绪日志mood, dimensions (JSON), timestamp
evolution_state成长状态level, xp, total_interactions

情绪系统公式

// 4 维度评分 (0-1)
comfort  = f(temp, humidity)    // 环境舒适度
interact = f(freq, sentiment)   // 交互情感
time_f   = f(hour, idle_min)    // 时间因子
anomaly  = f(sensor_err, face)  // 异常事件

// 加权合并
raw_mood = comfort×0.25 + interact×0.35 + time_f×0.25 + anomaly×0.15

// 平滑：EMA(α=0.3) + 连续 3 次一致才切换 + 最短 30s 持续
