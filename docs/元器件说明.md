# 元器件清单与说明

> 本文档列出 AI 桌宠项目（pi-pet-agent）中使用的所有电子元器件，包含型号、功能、接口、接线说明和驱动信息。

## 总览

本项目基于 **树莓派 4B**，共使用 **11 个主要元器件**（含可选 2 个），涵盖传感器、显示器、执行器和音频设备四大类。

| 2 | DHT11 温湿度传感器 | 传感器 | GPIO 17 | 1 |

| 4 | MQ-135 气体传感器 | 传感器 | MCP3008 CH0 | 1 |
| 5 | MCP3008 ADC 模数转换器 | 转换器 | SPI | 1 |

## 1. DS18B20 防水温度传感器

### 基本信息

| 属性 | 说明 |
|------|------|
| 型号 | DS18B20（防水探头封装） |
| 功能 | 精确测量环境温度，精度 ±0.5°C |
| 接口 | 1-Wire 协议 |
| 工作电压 | 3.0V ~ 5.5V |

### 接线方式

```
DS18B20        树莓派
──────────────────────
VCC (红)  ──→  3.3V (Pin 1)
DQ  (黄)  ──→  GPIO 4 (Pin 7)
GND (黑)  ──→  GND (Pin 6)
         ┌─── 4.7kΩ 上拉电阻 ───→ 3.3V
         └─── DQ 线
```

### 技术说明

- **必须接 4.7kΩ 上拉电阻**（连接 DQ 和 3.3V），否则无法正常通信
- 需在 `/boot/config.txt` 中添加 `dtoverlay=w1-gpio`
- 通过 Linux sysfs 文件系统读取：`/sys/bus/w1/devices/28-*`
- 无需额外 Python 库，直接读文件获取温度值

### 对应代码

- 测试函数：`hardware_test.py` → `test_ds18b20()`
- 传感器读取：`perception/sensors.py`

## 2. DHT11 温湿度传感器

### 基本信息

| 属性 | 说明 |
|------|------|
| 型号 | DHT11 |
| 功能 | 测量环境温度（±2°C）和湿度（±5%RH） |
| 接口 | 单总线数字信号 |
| 工作电压 | 3.3V ~ 5V |

### 接线方式

```
DHT11         树莓派
─────────────────────
VCC   ──→  3.3V (Pin 1)
DATA  ──→  GPIO 17 (Pin 11)
GND   ──→  GND (Pin 9)
        ┌─── 10kΩ 上拉电阻 ───→ 3.3V
        └─── DATA 线
```

### 技术说明

- **DATA 线需接 10kΩ 上拉电阻**到 3.3V
- Python 驱动：`adafruit-circuitpython-dht>=3.7`（主用），`Adafruit_DHT`（备选）
- DHT11 精度低于 DHT22，但满足课程设计需求

### 对应代码

- 测试函数：`hardware_test.py` → `test_dht11()`
- 配置定义：`config.py` → `PIN_DHT11 = 17`
- 传感器读取：`perception/sensors.py`

## 3. 光敏电阻模块（模拟光传感器）

### 基本信息

| 属性 | 说明 |
|------|------|
| 型号 | 光敏电阻模块（3 针：VCC/GND/SIG） |
| 功能 | 检测环境光照强度，输出模拟电压 |
| 接口 | 模拟输出 → MCP3008 CH1 |
| 工作电压 | 3.3V ~ 5V |

### 接线方式

```
光敏模块       树莓派 / MCP3008
──────────────────────────────
VCC  ──→  3.3V
GND  ──→  GND
SIG  ──→  MCP3008 CH1 (ADC 通道 1)
```

### 技术说明

- 树莓派没有内置 ADC，因此模拟信号需通过 MCP3008 转换后读取
- 读取的原始值（0-1023）映射到百分比（0-100%）

### 对应代码

- 测试函数：`hardware_test.py` → `test_light()`
- 配置定义：`config.py` → `ADC_SPI_CS = 7`、`ADC_CH_LIGHT = 1`

## 4. MQ-135 空气质量传感器

### 基本信息

| 属性 | 说明 |
|------|------|
| 型号 | MQ-135 |
| 功能 | 检测 NH₃、NOx、酒精、苯、烟雾、CO₂ 等有害气体 |
| 接口 | 模拟输出 → MCP3008 CH0 |
| 工作电压 | 5V（加热器需 5V 供电） |

### 接线方式

```
MQ-135         树莓派 / MCP3008
──────────────────────────────
VCC  ──→  5V（必须 5V，加热器需要）
GND  ──→  GND
AOUT ──→  MCP3008 CH0 (ADC 通道 0)
DOUT ──→  不接（数字输出未使用）
```

### 技术说明

- **首次使用需预热 24~48 小时**才能获得稳定读数，日常使用至少预热 2 分钟
- 气体浓度越高，模拟输出电压越高
- 检测到异常气体浓度时，LED 变红报警

### 对应代码

- 测试函数：`hardware_test.py` → `test_mq135()`
- 配置定义：`config.py` → `ADC_CH_MQ135 = 0`
- 报警联动：`hardware/led.py` → `gas_alert` 红色

## 5. MCP3008 模数转换器（ADC）

### 基本信息

| 属性 | 说明 |
|------|------|
| 型号 | MCP3008-I/P（DIP-16 封装） |
| 功能 | 8 通道 10-bit 模数转换，将模拟信号转为数字值 |
| 接口 | SPI 总线 |
| 工作电压 | 2.7V ~ 5.5V |
| 分辨率 | 10-bit（0~1023），参考电压 3.3V 时约 3.2mV/step |

### 接线方式

```
MCP3008          树莓派
────────────────────────
VDD / VREF  ──→  3.3V (Pin 1)
AGND / DGND ──→  GND (Pin 6)
CLK         ──→  GPIO 11 / SCLK (Pin 23)
DOUT (MISO) ──→  GPIO 9  (Pin 21)
DIN  (MOSI) ──→  GPIO 10 (Pin 19)
CS/SHDN     ──→  GPIO 7  (Pin 26) — SPI CE1

CH0 (通道0) ──←  MQ-135 AOUT
CH1 (通道1) ──←  光敏电阻模块 SIG
```

### 技术说明

- MCP3008 和 SSD1306 OLED **共享 SPI0 总线**（GPIO 9/10/11）
- 通过不同的片选信号区分设备：OLED 用 CE0（GPIO 8），MCP3008 用 CE1（GPIO 7）
- Python 库：`spidev`

### 对应代码

- 配置定义：`config.py` → `ADC_SPI_CS = 7`

## 6. SSD1306 OLED 显示屏

### 基本信息

| 属性 | 说明 |
|------|------|
| 型号 | SSD1306（0.96 英寸，7 针 SPI 版） |
| 功能 | 128×64 像素单色显示，用于表情、文字、数据 |
| 接口 | SPI 总线 |

### 接线方式

```
SSD1306         树莓引脚

DIN  ──→  GPIO 10 / MOSI (Pin 19) — SPI 共享
```

- 内置 7 种表情像素画（happy/sad/sleepy/angry/surprised/love/neutral）

### 对应代码

- 工具封装：`tools/oled_tool.py`

## 7. RGB LED（共阳极）

### 基本信息

| 属性 | 说明 |
|------|------|
| 型号 | 5mm 共阳极 RGB LED |
| 功能 | 通过不同颜色表达桌宠情绪状态 |
| 接口 | 3 路 GPIO PWM（1000 Hz） |
| 工作电压 | 约 2.0V ~ 3.3V（每路需串联限流电阻） |

### 接线方式

```
RGB LED          树莓派
────────────────────────
公共阳极 (最长脚) ──→  3.3V
R (红，330Ω)    ──→  GPIO 27 (Pin 13)
G (绿，330Ω)    ──→  GPIO 22 (Pin 15)
B (蓝，330Ω)    ──→  GPIO 5  (Pin 29)
```

### 情绪颜色映射

| 情绪 | 颜色 | 说明 |
|------|------|------|
| happy（开心） | 暖黄色 | `RGB(255, 180, 0)` |
| curious（好奇） | 青色 | `RGB(0, 255, 255)` |
| sleepy（困倦） | 暗紫色 | `RGB(80, 0, 120)` |
| alert（警觉） | 红色 | `RGB(255, 0, 0)` |
| chatty（健谈） | 绿色 | `RGB(0, 255, 0)` |
| calm（平静） | 蓝色 | `RGB(0, 100, 255)` |
| lonely（孤单） | 靛蓝色 | `RGB(75, 0, 130)` |

### 技术说明

- 共阳极接法：GPIO 输出 LOW = 亮，HIGH = 灭（电平取反）
- 每路串联 **330Ω 限流电阻**保护 LED
- 使用 `RPi.GPIO` 的 PWM 功能，频率 1000Hz

### 对应代码

- 驱动文件：`hardware/led.py`（111 行，含情绪颜色映射和闪烁）
- 工具封装：`tools/led_tool.py`
- 配置定义：`config.py` → `PIN_LED_R = 27`、`PIN_LED_G = 22`、`PIN_LED_B = 5`

## 8. 有源蜂鸣器

### 基本信息

| 属性 | 说明 |
|------|------|
| 型号 | 5V 有源蜂鸣器（自激式） |
| 功能 | 发出提示音，支持多种音效模式 |
| 接口 | GPIO 数字输出（HIGH = 响，LOW = 停） |
| 工作电压 | 3.3V ~ 5V |

### 接线方式

```
蜂鸣器           树莓派
────────────────────────
VCC / I/O  ──→  GPIO 12 (Pin 32)
GND        ──→  GND (Pin 34)
```

### 音效模式

| 模式 | 效果 |
|------|------|
| beep | 单次短鸣（0.1s） |
| reminder | 3 次短鸣提醒 |
| welcome | 欢迎音 |
| alarm | 5 次快速警报 |
| level_up | 升调三音旋律（523→659→784 Hz） |

### 技术说明

- 有源蜂鸣器内置振荡电路，通电即响，无需提供频率信号
- 选择 GPIO 12 而非 GPIO 18，因为 GPIO 18 被 I2S 麦克风的 BCLK 占用
- 使用 `RPi.GPIO` 简单的数字 HIGH/LOW 控制

### 对应代码

- 驱动文件：`hardware/buzzer.py`（86 行，异步支持）
- 工具封装：`tools/buzzer_tool.py`
- 配置定义：`config.py` → `PIN_BUZZER = 12`

## 10. INMP441 I2S 数字麦克风（×2，立体声）

### 基本信息

| 属性 | 说明 |
|------|------|
| 型号 | INMP441 I2S MEMS 数字麦克风模块 |
| 功能 | 立体声录音，用于语音指令识别 |
| 接口 | I2S 数字音频 |
| 数量 | 2 个（1 左声道 + 1 右声道） |

### 接线方式

```
INMP441 (左声道)     树莓派
───────────────────────────
VDD  ──→  3.3V
GND  ──→  GND
SCK  ──→  GPIO 18 (Pin 12) — I2S BCLK
WS   ──→  GPIO 19 (Pin 35) — I2S LRCK
SD   ──→  GPIO 20 (Pin 38) — I2S DIN
L/R  ──→  GND（选择左声道）

INMP441 (右声道)     树莓派
───────────────────────────
VDD  ──→  3.3V
GND  ──→  GND
SCK  ──→  GPIO 18（共享）
WS   ──→  GPIO 19（共享）
SD   ──→  GPIO 20（共享同一条数据线）
L/R  ──→  3.3V（选择右声道）
```

### 技术说明

- 两个麦克风共享 SCK/WS/SD 三根线，通过 L/R 引脚区分左右声道
- 需在 `/boot/config.txt` 中添加 `dtoverlay=i2s-mmic`
- 录音命令：`arecord -D dmic_sv -c2 -r 16000 -f S16_LE -t wav`
- 支持 VAD（语音活动检测）：`webrtcvad>=2.0.10`
- 语音流程：VAD 能量检测 → 云端 STT API → 文本 → Agent 处理

### 对应代码

- 麦克风驱动：`perception/microphone.py`（141 行）
- 语音监听管线：`perception/listener.py`（309 行，完整 VAD + STT 流程）
