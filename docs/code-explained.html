二、入口与配置

main.py 真实逻辑
程序主入口，启动 Web 服务
• 配置 logging（输出到终端 + 文件 data/logs/system.log）
• 调用 uvicorn.run("web.app:app") 启动 FastAPI 服务
• reload=True 开发时自动重载
config.py 真实逻辑
全局配置常量，所有模块共用
• 路径：DATA_DIR / FACES_DIR / SNAPSHOTS_DIR / MODELS_DIR 等
• LLM：DeepSeek API Key / Base URL / Model / Temperature
• 硬件引脚：DHT11(4) / LED_R(27) / LED_G(22) / LED_B(5) / Buzzer(18) 等
• 情绪系统：EMA α=0.3 / 连续 3 次才切换 / 最短 30s
• 记忆系统：工作记忆 10 轮 / 短期 7 天 / 向量检索 top 5
• Web：host 0.0.0.0 / port 8000

三、Web 服务层

web/app.py 真实逻辑
FastAPI 应用创建 + 生命周期管理
• lifespan 启动：初始化 SQLite 数据库 → 启动传感器轮询后台线程（每 10 秒）
• lifespan 关闭：设置停止事件 → 等待线程退出
• 挂载 6 个路由：/api/chat / /api/sensors / /api/camera / /api/memory / /api/reminders / /api/status
• 挂载 /static 静态文件目录
• GET / 返回 index.html 主页
web/routers/sensors.py 真实逻辑
传感器数据 API，已接入 SQLite
• GET /api/sensors → 从 DB 读取最新一条传感器数据 + 舒适度评分
• GET /api/sensors/history?range=24h → 支持 1h/6h/24h/7d 四档降采样查询
• 降采样逻辑：按 step_seconds 过滤，避免返回过多数据
web/routers/chat.py 模拟数据
AI 对话接口
• POST /api/chat → 接收 user_input，返回 echo 回复（后续接入 DeepSeek LLM）
• POST /api/chat/stream → SSE 流式返回，每 token 延迟 80ms 模拟打字效果
web/routers/camera.py 模拟数据
摄像头接口
• GET /api/camera/stream → MJPEG 流占位（后续接入 OpenCV）
• POST /api/camera/capture → 拍照分析占位
web/routers/memory.py 模拟数据
记忆查询接口
• GET /api/memory → 记忆列表（后续接入 SQLite + ChromaDB）
• GET /api/memory/search?q=xxx → 语义搜索
web/routers/reminders.py 模拟数据
提醒 CRUD 接口
• GET /api/reminders → 列表
• POST /api/reminders → 创建
• DELETE /api/reminders/{id} → 删除
web/routers/status.py 模拟数据
宠物状态接口
• GET /api/status → 情绪 + 等级 + 经验
• GET /api/evolution → 进化状态

四、数据库与记忆系统

memory/database.py 真实逻辑
SQLite 初始化与连接管理
• init_db() — 启动时执行 schema.sql 建表，启用 WAL 模式
• get_conn() — 上下文管理器，自动 commit/rollback，Row Factory 返回字典
memory/sensor_manager.py 真实逻辑
传感器数据读写 + 历史查询
• read_and_store() — 读取传感器 → 写入 sensor_readings 表（每 10 秒调用）
• get_latest() — 返回 DB 中最新一条数据 + 舒适度评分
• get_history(hours, step_seconds) — 按时间范围查询 + 降采样过滤
• _calc_comfort() — 基于温湿度计算舒适度（最优：20-26°C, 40-60%RH）
memory/schema.sql 真实逻辑
SQLite 数据库表定义
• sensor_readings — 温度/湿度/光照，每 10 秒一条
• interactions — 对话记录（来源/输入/回复/情绪/工具）
• short_term_memory — 每日摘要（统计/话题/温度范围）
• long_term_memory — 长期记忆（类别/重要度/ChromaDB 向量 ID）
• reminders — 提醒规则（单次/周期/条件触发）
• mood_log — 情绪变化日志（情绪/分数/影响因素 JSON）
• evolution_state — 进化状态（等级/经验值/性格 JSON）
memory/manager.py 接口骨架
MemoryManager 统一记忆接口（待实现）
• store(type, content) / recall(query) / get_recent_context()
• get_short_term() / reflect() — 后续接入 ChromaDB 向量检索

五、Agent 状态机

agent/state.py 真实逻辑
PetState 类型定义（TypedDict）
• messages — 对话历史
• sensor_data — 当前传感器快照
• voice_text / image_desc — 多模态输入
• recalled_memories — 检索到的记忆
• mood — 当前情绪状态
• oled_text / oled_expression — 硬件输出指令
• next_action — 下一步决策
agent/graph.py 接口骨架
LangGraph 状态机图定义（待实现）
• 图结构：START → Perceive → Recall → Think → Decide → [Act | Tools | Reflect] → END
• Decider 节点根据 Think 输出路由到不同分支
agent/prompt.py 真实逻辑
System Prompt 构建器
• MOOD_PROMPT_MAP — 7 种情绪对应的 prompt 片段
• build_system_prompt(mood, memories) — 组装完整 system prompt（角色设定 + 情绪 + 记忆）

六、感知层

perception/sensors.py 真实逻辑
传感器统一读取接口
• HARDWARE_AVAILABLE 标志位，自动判断是否在树莓派上
• 有硬件 → 调用 DS18B20 + DHT11 + BH1750 驱动
• 无硬件 → 返回 mock 随机数据（开发环境用）
perception/camera.py 接口骨架
摄像头采集接口（待接入 OpenCV）
• capture() — 拍照返回图像
• analyze() — 图像分析（人脸检测/场景理解）
• stream() — MJPEG 帧生成器
perception/speech.py 接口骨架
语音识别接口（待接入 Whisper）
• listen() — 录音
• transcribe() — Whisper 语音转文字

七、硬件驱动

hardware/oled.py 接口骨架
SSD1306 OLED 显示屏驱动（I2C）
• show_expression(name) — 显示 128×64 表情位图
• show_text(text) — 滚动文字
• show_data(sensor) — 显示传感器数值
• clear() — 清屏
hardware/led.py 接口骨架
RGB LED 驱动（GPIO PWM）
• MOOD_COLORS — 7 种情绪对应颜色（happy→金色, alert→红色 等）
• set_mood(mood) — 根据情绪设置 LED 颜色
• set_color(r,g,b) — 直接设置颜色
hardware/buzzer.py 接口骨架
蜂鸣器驱动（GPIO PWM）
• beep(frequency, duration) — 基础发声
• reminder_sound() / welcome_sound() / level_up_sound()
hardware/speaker.py 接口骨架
USB 音响驱动（TTS 语音合成）
• speak(text) — edge-tts 合成 → 播放
• play_wav(path) / play_mp3(path) — 播放音频文件

八、调度器

scheduler/reminder.py 接口骨架
APScheduler 提醒调度（待实现）
• add_once(trigger_at, content) — 单次定时提醒
• add_cron(cron_expr, content) — 周期提醒（如每天 8:00）
• add_interval(seconds, content) — 间隔提醒（如每小时喝水）
• remove(job_id) / list_jobs()

九、前端文件

web/static/index.html
单页应用主页，5 个 Tab 面板
• Header：房子图标 + 名称 + 时间/祝福语 + Tab 导航 + 通知/设置
• AI 对话：左侧消息流 + 右侧历史记录栏 + 输入框
• 摄像头：视频区 + 控制按钮 + 右侧分析/快照
• 传感器：4 张数据卡片 + 温度/湿度/情绪图表
• 记忆：搜索栏 + 长期记忆列表 + 短期摘要
• 提醒：提醒列表 + 添加表单 + 已过期区
web/static/style.css
暖色护眼主题，全部使用 CSS Custom Properties 令牌
• 设计令牌（surface/text/accent/semantic/shadow/motion）
• 组件样式（header/tabs/buttons/inputs/chat/camera/sensors/memory/reminders）
• 响应式（1024px / 768px 断点）+ reduced-motion 适配
web/static/app.js
前端交互逻辑
• initTabs() — Tab 切换 + ARIA 状态同步
• initChat() — 发送消息 + 模拟回复 + 滚动到底部
• initCharts() — Chart.js 温度/湿度趋势图 + Canvas 情绪时间线
• showToast() — 通知气泡系统
• rotateGreeting() — 祝福语每 30 秒随机切换
• updateClock() — 实时时钟显示
