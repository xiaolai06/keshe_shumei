设计令牌

Design Tokens — 一切视觉决策的源头

本项目采用 CSS Custom Properties 管理全部视觉变量。所有组件仅引用令牌（--token-name），禁止硬编码色值、字号、间距。修改一处变量即全局生效。

令牌架构

层级用途示例
Primitive基础色值、尺寸#6e8cff、14px
Semantic语义化用途映射--accent、--text-2
Component组件级覆盖--btn-padding、--card-radius
💡
当前版本使用 Semantic 层，组件层在具体实现时按需定义。所有变量定义在 :root 中，支持后续扩展暗/亮主题切换。

动效规范

Motion & Animation

动效令牌

Token值用途
--dur-fast120msHover 反馈、Focus ring
--dur-normal200msTab 切换、展开/折叠
--dur-slow350ms侧边栏、弹窗进出
--ease-outcubic-bezier(.25, 1, .5, 1)所有进入动画
--ease-springcubic-bezier(.34, 1.56, .64, 1)强调反馈（按钮 active）

动效原则

  • 1

    只动 compositor 属性

    transform 和 opacity 仅此两项。不动画 width、height、margin、padding。

  • 2

    动效传达状态

    Tab 切换、Toast 进出、骨架屏 shimmer。装饰性动画一律不加。

  • 3

    尊重 prefers-reduced-motion

    所有动画在 prefers-reduced-motion: reduce 下设为 0ms 或隐藏。

响应式设计

Responsive Breakpoints

断点

断点值设备布局变化
Desktop> 1024pxPC、平板横屏完整布局：Grid 双栏、侧栏可见
Tablet768~1024px平板竖屏缩小间距，卡片 2 列，聊天历史可折叠
Mobile< 768px手机单栏、Tab 栏底部固定、聊天历史隐藏

响应式规则

  • 1

    聊天区

    PC: 1fr 320px 双栏 → 移动端: 单栏，历史通过 "查看历史" 按钮全屏覆盖

  • 2

    摄像头

    PC: 视频 + 分析面板双栏 → 移动端: 视频全宽，面板在下方

  • 3

    传感器图表

    PC: 湿度 + 情绪并排 → 移动端: 垂直堆叠

  • 4

    记忆

    PC: 长期 + 短期双栏 → 移动端: Tab 切换两个视图

  • 5

    Tab 栏

    PC: 顶部横向 → 移动端: 底部固定，图标 + 短文字
